Agno

优势
  • Agent/Team/Workflow 三层抽象覆盖绝大多数场景
  • 内置生产级部署（无状态 Runtime + AgentOS）
  • 100+ 开箱即用集成（工具、向量库、模型）
  • Memory/Knowledge/Learning 内置且完善
  • Guardrails + Human-in-the-Loop 原生支持
  • 从单 Agent 到多 Agent 系统平滑扩展
劣势
  • 无自动 Prompt 优化能力（纯手工 instruction）
  • 前身 PhiData 品牌迁移可能带来混淆
  • AgentOS 平台锁定风险
  • 相比 LangChain 社区生态仍小

DSPy

优势
  • 自动 Prompt/权重优化是独有杀手级特性
  • 声明式编程让 LM 程序更可维护、可复用
  • 模型无关：换模型只需 re-compile
  • Stanford 学术背景，理论基础扎实
  • 内置评估系统，数据驱动迭代
  • MCP 协议支持
劣势
  • 学习曲线较陡（需理解优化器思维）
  • 不是完整的 Agent 框架（无 Memory/部署/监控）
  • 多 Agent 编排能力有限
  • 生产部署需自行解决
  • 偏学术，企业级文档和最佳实践较少

pi-mono

优势
  • 专为复杂企业场景设计（招投标分析等）
  • 多层 Agent 架构（6 层），任务分解能力强
  • 并发 LLM 调用，性能优化
  • 已在矩阵起源生产环境验证
  • 可针对业务深度定制
劣势
  • 非开源，无法复用
  • 无公开文档和社区支持
  • 与特定公司业务绑定
  • 人才迁移成本高

LangChain

优势
  • 最大的社区和生态系统
  • 几乎所有模型和工具都有集成
  • LangGraph 解决了复杂编排需求
  • LangSmith 提供完整可观测性
  • 文档和教程最丰富
  • Python + JS 双语言支持
劣势
  • 抽象层过多，"overengineered"
  • API 频繁变更，升级成本高
  • 性能开销较大
  • 简单任务也需大量 boilerplate
  • 核心功能分散到多个包（langchain-core/community/...）

与 LangChain 的核心差异

Agno vs LangChain

Agno 是 "Agent Runtime"，LangChain 是 "LLM Framework"。Agno 自带部署运行时和管控平面，不需要拼装 LangServe + LangSmith + LangGraph 多个产品。API 更简洁，从单 Agent 到 Team 到 Workflow 一套代码搞定。但 LangChain 生态更大，第三方集成更多。

DSPy vs LangChain

本质定位不同。DSPy 解决的是 "如何让 LLM 输出更好"（自动优化 prompt/权重），LangChain 解决的是 "如何组装 LLM 应用"（Chain/Agent 编排）。DSPy 可以和 LangChain 互补使用 — 用 DSPy 优化核心 LM 调用，用 LangChain 做外围集成。DSPy 反对手写 prompt，LangChain 围绕 prompt template 展开。

pi-mono vs LangChain

pi-mono 是针对特定企业场景的深度定制框架，强调多层架构和并发性能。LangChain 是通用框架，覆盖面广但在复杂企业场景中可能需要大量定制。pi-mono 的分层 Agent 设计思路可以用 LangGraph 部分实现，但后者需要更多胶水代码。
