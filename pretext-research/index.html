Pretext: Zero-Reflow 文本排版引擎

一个纯 TypeScript 库，用 Canvas measureText 取代 DOM 回流，让多行文本的高度预测和换行计算变成纯数学运算。两天内获得 6,000+ Stars。

作者 Cheng Lou (chenglou) 语言 TypeScript 协议 MIT 版本 v0.0.3 依赖 零运行时依赖
01

它是什么

Pretext 是一个浏览器端的多行文本测量与布局库，发布为 npm 包 @chenglou/pretext。它的核心洞察是：浏览器的 Canvas measureText() API 可以作为字体引擎的"真相来源"（ground truth），完全绕过昂贵的 DOM 回流操作。

在传统 Web 开发中，每次需要知道"这段文字有多高"或"在哪里换行"，都必须调用 getBoundingClientRect() 或读取 offsetHeight，触发浏览器的 layout reflow。当页面有数百个组件时，这会导致严重的性能瓶颈。

Pretext 将这个问题分解为两个独立的阶段：一次性的文本分析（expensive），和纯算术的布局计算（cheap）。这意味着窗口 resize 时不需要任何 DOM 读取。

02

两阶段架构

Pretext 的架构优雅地分离了"昂贵但只做一次"和"便宜且频繁调用"两种操作。这是理解整个库的关键。

prepare(text, font) — 预处理阶段
一次性工作。规范化空白字符、用 Intl.Segmenter 分割词段、应用 CJK/阿拉伯语/Emoji 的特殊规则、用 Canvas measureText() 测量每个词段宽度、缓存所有度量信息。约 18ms / 500段文字。返回不透明的 PreparedText handle。
layout(prepared, maxWidth, lineHeight) — 布局阶段
热路径。纯算术运算，零 DOM 读取、零 Canvas 调用。遍历缓存的宽度数据，累加比较，返回行数和高度。约 0.09ms / 500段文字。每次 resize 只需调用这个函数。
layoutNextLine(prepared, cursor, maxWidth) — 逐行流式布局
进阶 API。每次调用返回一行的文本、宽度和光标位置。每行可以指定不同的可用宽度，实现文字环绕任意形状障碍物、多栏连续流动等高级排版。
walkLineRanges(prepared, maxWidth, onLine) — 批量几何计算
高性能批量 API。获取每行的宽度和光标位置，但不构建文本字符串。适用于 shrinkwrap（二分搜索最紧凑宽度）等场景。
设计哲学：opaque fast path + rich escape hatch。prepare() 返回不透明 handle 防止 API 僵化，prepareWithSegments() 是给高级用户的逃生通道。
03

核心能力

Pretext 不只是一个测量工具，它是一个完整的文本布局引擎，覆盖了从简单高度预测到复杂编辑排版的全部场景。

DOM-Free 测量
零回流 / 零 getBoundingClientRect
用 Canvas measureText 替代所有 DOM 度量操作。组件级 UI 中消除 layout thrashing，resize 时仅执行纯数学运算。
全语言支持
CJK / Arabic / Thai / Emoji
基于 Intl.Segmenter 的国际化分词。CJK 逐字换行 + 禁则处理（kinsoku），阿拉伯语双向文本，东南亚语系（泰/高棉/缅甸），Emoji ZWJ 序列。
变宽排版
layoutNextLine API
每行可以指定不同的可用宽度。文字可以像水一样流过任意形状的障碍物，实现杂志般的多栏流动和 obstacle-aware 路由。
Shrinkwrap 测量
walkLineRanges + 二分搜索
找到保持相同行数的最窄容器宽度。CSS fit-content 无法做到这一点。适用于聊天气泡、卡片等需要紧凑排列的场景。
CSS 行为匹配
white-space: normal / pre-wrap
精确匹配浏览器的文本换行行为。支持 overflow-wrap: break-word，trailing spaces hang past line edge，soft hyphen 断行时显示连字符。
Emoji 自动校正
Canvas vs DOM 宽度差异
Chrome/Firefox 在 macOS 上 Canvas 测量 emoji 比 DOM 更宽。Pretext 自动检测并校正这个差异，每字体只需一次 DOM 读取。

优势与局限

优势

  • 零 DOM 回流的文本高度预测
  • 完整的国际化换行支持
  • 变宽排版 / 障碍物环绕
  • 零运行时依赖，纯 TypeScript
  • 三大浏览器 100% 精度测试
  • Shrinkwrap 聊天气泡优化

局限

  • 仅支持 white-space: normal / pre-wrap
  • 不支持 break-all / keep-all 等 CSS 属性
  • system-ui 字体在 macOS 上不安全
  • Tab 固定为 8 字符宽度
  • 服务端渲染尚在规划中
  • 版本 v0.0.3，API 可能变动

结论

一个解决了 30 年老问题的优雅方案

Pretext 不是一个增量改进——它从根本上重新思考了 Web 上文本排版的工作方式。通过将测量与布局分离为两个独立阶段，它让一个长期困扰前端开发者的性能瓶颈变得不再存在。

项目的爆发式增长（48 小时 6K Stars）反映了三件事：需求真实存在、解决方案足够优雅、作者的技术信誉深厚。Cheng Lou 用 react-motion 改变了 React 动画的思维方式，用 ReasonML 证明了类型系统在前端的价值。现在，Pretext 正在改变我们思考文本排版的方式。

对于需要精细文本布局的应用——编辑器、阅读器、消息应用、数据可视化、虚拟列表——Pretext 很可能会成为基础设施级别的存在。它现在是 v0.0.3，但底层的工程质量（7,680 项跨浏览器测试、详尽的语料库验证、生产级的性能数据）已经远超这个版本号暗示的成熟度。
