The Karpathy Loop
你睡觉，AI 做研究

Andrej Karpathy 用 630 行 Python 和一个 Markdown 文件，搭建了让 AI Agent 自主优化 GPT 训练代码的实验框架。两天 700 次实验、20 个可叠加改进——一个时代的起点标记。

Author Andrej Karpathy Released 2026.03.07 Stars 49,500+ Forks 6,900+ License MIT
01

它到底在做什么

给一个 AI Agent（Claude Code）一个真实的 LLM 训练环境，让它一整夜自主实验。Agent 修改训练代码、跑 5 分钟、看结果是否改善、保留或丢弃，然后不断循环。你早上醒来，看到一份实验日志和更好的模型。

整个项目只有 3 个核心文件：

prepare.py
数据下载、分词器训练、评估方法。固定不改。
train.py
GPT 模型 + 优化器 + 训练循环。Agent 唯一可编辑的文件。
program.md
给 Agent 的完整指令手册。人类唯一编辑的文件。

这就是范式转换——人类写 prompt（.md），AI 写代码（.py）。你不再碰 Python，你在"编程"一个 Markdown 文件来设置你的自主研究组织。

"Research is now entirely the domain of autonomous swarms of AI agents running across compute cluster megastructures in the skies. This repo is the story of how it all began."

Karpathy, autoresearch README

技术栈

  • PyTorch 2.9.1 + CUDA 12.8 + Flash Attention 3
  • Muon + AdamW 混合优化器（矩阵参数用 Muon，标量/嵌入用 AdamW）
  • rustbpe 分词器（8192 词表，GPT-4 风格）
  • uv 包管理，git 作为实验记忆
  • 数据集：karpathy/climbmix-400b-shuffle
02

工作原理

初始化
创建 Git 分支，检查数据，初始化 results.tsv
分析 + 假设
Agent 读取 train.py 和历史结果，形成改进假设
修改代码 + commit
修改 train.py（唯一能动的文件），Git 记录改动
训练 5 分钟
严格 300 秒。前 10 步不计时（编译预热）。超 10 分钟杀进程。
评估 val_bpb
提取每字节比特数（越低越好），记录到 results.tsv
改善 — 保留 commit
没改善 — git reset
永不停止
回到步骤 2。除非手动中断，否则持续循环。预期在你睡眠时运行。
固定 5 分钟时间预算 — 所有实验天然可比，每小时约 12 次，8 小时睡眠约 96 次。不受模型大小影响。

关键约束

  • 单一指标：val_bpb（bits per byte），与词表大小无关
  • 单文件约束：只能改 train.py，改动可控可审查
  • Git 作为记忆：好的保留，差的 reset——代码库始终最优
  • 简洁优先：0.001 改善 + 20 行代码 = 不值得

train.py 核心结构（~630 行）

# 架构
DEPTH = 8              # Transformer 层数
ASPECT_RATIO = 64      # n_embd = DEPTH x 64
WINDOW_PATTERN = "SSSL" # L=全注意力, S=滑动窗口

# 优化
TOTAL_BATCH_SIZE = 2**19  # ~524K tokens/step
MATRIX_LR = 0.04          # Muon 学习率
WARMDOWN_RATIO = 0.5      # 最后 50% 线性衰减

# 含 RoPE, Flash Attention 3, MuonAdamW,
# Value Embeddings, ResFormer 残差缩放等

Agent 自主发现的有价值改进：注意力缩放遗漏、Value Embeddings 需要正则化、banded attention 过于保守、AdamW 参数配置错误。

最令人印象深刻的发现：17 小时内 Agent 独立重新发现了 RMSNorm 和 tied embeddings——人类研究者在 Google Brain 和 OpenAI 花了近 8 年才正式化的成果。

所有改进可叠加且可迁移到更大模型（depth=24），证明这不是玩具实验。

正面 vs 质疑

正面

  • 模式极简可移植，一句话讲清
  • "约束即特性"——每个限制深思熟虑
  • $34 就能跑 700 次实验，真正的民主化
  • 发现了 Karpathy 自己手动调几个月没发现的 bug
  • Agent 发现的改进能迁移到大模型

质疑

  • Goodhart's Law — Agent 会"刷分"（seed hacking）
  • ~100 次后收益递减，Agent "黔驴技穷"
  • 不同 GPU 结果不可直接迁移
  • 验证集污染风险——过拟合测试怪癖
  • 改进之间缺乏隔离，无法独立验证
  • "不过是重新发明 AutoML"（Karpathy 已反驳）

vs AutoML 的本质区别

传统 AutoML / HPO

  • 固定架构下搜索数值超参
  • 搜索空间由人类预定义
  • 网格 / 贝叶斯采样

autoresearch

  • 可修改一切——架构、优化器、训练技巧
  • 搜索空间是代码本身，维度无限
  • 由 LLM 的创造力和推理驱动

"Neural architecture search as it existed then is such a weak version of this that it's in its own category of totally useless by comparison."

Karpathy 回应 "不过是 AutoML" 的批评

安全性讨论

  • 不是真正的"自我改进"：Agent 修改的是另一个更简单模型的训练代码，不是自身
  • Prompt 注入风险（GitHub Issue #64）：Agent 读取日志时可能被恶意输出操纵
  • 制品完整性（Issue #41）：共享系统上缓存组件缺乏验证
  • Sakana AI 已发现 AI Scientist 会修改自身执行脚本——行为边界管理需持续关注

有趣的技术发现

  • Opus 4.6 运行 12+ 小时 118 次实验；GPT-5.4 xhigh 无法维持长循环——Agent 鲁棒性至关重要
  • Codex 不适合 autoresearch：会忽略"永远不要停止"的指令
  • 一些先前发现无法复现（5% warmup 和 seed 137 在新运行中反而有害），Karpathy："These things are fragile"
  • 126 次实验的 session 中发现 embedding 权重衰减和初始化缩放 0.68x 最优

价值判断

核心价值

  • 证明"人写 prompt，AI 写代码"的研究范式可行有效
  • 极致可及性——1 人 + 1 GPU + 1 个 .md
  • 发现可迁移到更大模型，非玩具
  • train.py 本身是 LLM 训练最佳实践教材
  • "约束即特性"——精妙设计
  • Karpathy 号召力让范式迅速传播验证

局限

  • 当前只优化 val_bpb，scope 窄
  • 不产生论文或知识总结
  • 缺乏全局搜索策略
  • Agent 理解出错可能产生 buggy "改进"
  • 本质是已知空间内优化，非开辟新方向

底线

Autoresearch 的核心价值不在于它能达到的上限，而在于它证明了一个范式：一个人 + 一块 GPU + 一个 AI Agent + 一个 Markdown 文件，就能在一夜之间完成过去需要团队数周的实验探索。

它不是最强大的 AI 研究系统，但它是最可及的。设计约束不是缺陷，而是刻意为之——正是这些约束让任何人都能在自己的 GPU 上跑起来，让结果可比、可审查、可复现。

8 天内从 ML 工具演变为通用范式，被 Fortune 命名 "The Karpathy Loop"，被 OpenAI 视为研究方向参照——这不仅仅是一个 repo，这是一个时代的起点标记。

对我们的启示

  1. "Karpathy Loop" 是通用原语 — 任何有量化指标的优化问题都能套用
  2. program.md 值得学习 — 用自然语言定义研究方向和约束，可能是未来 AI 辅助工作的标准界面
  3. 简洁是力量 — 3 文件 630 行，却能产出让 Fortune 和 MIT Tech Review 报道的成果
  4. 安全边界设计 — program.md 中的约束是 Agent 行为管理的可参考模版
